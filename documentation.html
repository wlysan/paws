Guia Completo de Documentação da API

Introdução

Esta documentação fornece uma visão detalhada de todos os endpoints de API disponíveis em nosso sistema. Cada endpoint é explicado com seus parâmetros, exemplos de uso e possíveis respostas.

Autenticação

Todos os endpoints, exceto registro de usuário, requerem autenticação via token de sessão:

  1. Registre-se usando o endpoint de registro
  2. Faça login para receber um token de sessão
  3. Inclua o token no cabeçalho Authorization: Bearer {token}
Importante: Os tokens de sessão têm tempo de expiração. Use o endpoint de renovação para manter a sessão ativa.

Domínios de API

1. Registro de Usuário

POST /registrarUsuario

Registra um novo usuário no sistema

Parâmetros:
  • nome (string, obrigatório): Nome completo do usuário
  • email (string, obrigatório): Endereço de e-mail único
  • senha (string, obrigatório): Senha com no mínimo 8 caracteres
  • tipo_usuario (string, opcional): Tipo de usuário (padrão: 'cliente')
  • telefone (string, opcional): Número de telefone do usuário
// Exemplo de requisição { "nome": "Maria Silva Santos", "email": "maria.silva@exemplo.com", "senha": "SenhaForte@2025", "tipo_usuario": "cliente", "telefone": "+55 (11) 98765-4321" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Usuário registrado com sucesso", "id_usuario": 9876, "email": "maria.silva@exemplo.com" }
GET /listarUsuarios

Lista todos os usuários registrados (requer permissão de administrador)

Parâmetros de Consulta:
  • pagina (integer, opcional): Número da página (padrão: 1)
  • limite (integer, opcional): Número de resultados por página (padrão: 10)
  • tipo_usuario (string, opcional): Filtrar por tipo de usuário
// Resposta de sucesso { "status": "sucesso", "total_usuarios": 50, "pagina_atual": 1, "usuarios": [ { "id": 9876, "nome": "Maria Silva Santos", "email": "maria.silva@exemplo.com", "tipo_usuario": "cliente", "data_registro": "2025-02-11T19:10:00Z" }, { "id": 9877, "nome": "João Pereira", "email": "joao.pereira@exemplo.com", "tipo_usuario": "cliente", "data_registro": "2025-02-10T15:30:00Z" } ] }
GET /buscarUsuario

Busca informações de um usuário específico

Parâmetros:
  • email (string, obrigatório): E-mail do usuário
// Exemplo de requisição { "email": "maria.silva@exemplo.com", } // Resposta de sucesso { "status": "sucesso", "usuario": { "id": 9876, "nome": "Maria Silva Santos", "email": "maria.silva@exemplo.com", "tipo_usuario": "cliente", "telefone": "+55 (11) 98765-4321", "data_registro": "2025-02-11T19:10:00Z", "ultimo_login": "2025-02-11T19:15:00Z" } }
PUT /atualizarUsuario

Atualiza informações do usuário

Parâmetros:
  • email (string, obrigatório): E-mail do usuário
  • nome (string, opcional): Novo nome do usuário
  • telefone (string, opcional): Novo número de telefone
  • tipo_usuario (string, opcional): Atualizar tipo de usuário
// Exemplo de requisição { "email": "maria.silva@exemplo.com", "nome": "Maria Silva Santos Oliveira", "telefone": "+55 (11) 99999-8888", "tipo_usuario": "premium" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Usuário atualizado com sucesso", "usuario_atualizado": { "nome": "Maria Silva Santos Oliveira", "telefone": "+55 (11) 99999-8888", "tipo_usuario": "premium" } }
PUT /desativarUsuario

Desativa a conta do usuário

Parâmetros:
  • email (string, obrigatório): E-mail do usuário
  • motivo (string, opcional): Motivo da desativação
// Exemplo de requisição { "email": "maria.silva@exemplo.com", "motivo": "Solicitação do usuário" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Usuário desativado com sucesso", "data_desativacao": "2025-02-11T19:20:00Z" }
PUT /deletarUsuario

Reativa uma conta de usuário previamente desativada

Parâmetros:
  • email (string, obrigatório): E-mail do usuário
// Exemplo de requisição { "email": "maria.silva@exemplo.com" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Usuário reativado com sucesso", "data_reativacao": "2025-02-11T19:25:00Z" }

Deleta a conta do usuário (remoção parcial)

Parâmetros:
  • email (string, obrigatório): E-mail do usuário
  • motivo (string, opcional): Motivo da deleção
// Exemplo de requisição { "email": "maria.silva@exemplo.com", "motivo": "Solicitação do usuário" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Usuário deletado com sucesso", "data_delecao": "2025-02-11T19:30:00Z" }

Verifica se a sessão do usuário está válida

Parâmetros:
  • email (string, obrigatório): Email do usuário
// Exemplo de requisição { "email": "usuario@exemplo.com" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Sessão válida." }

Lista o histórico de login do usuário

Parâmetros:
  • email (string, obrigatório): Email do usuário
// Exemplo de requisição { "email": "usuario@exemplo.com" } // Resposta de sucesso { "status": "sucesso", "dados": [ { "data_hora": "2025-02-12 10:30:00", "endereco_ip": "192.168.1.1", "informacao_dispositivo": "Mozilla/5.0...", "sucesso": true, "motivo_falha": null } ] }

Altera a senha do usuário

Parâmetros:
  • email (string, obrigatório): Email do usuário
  • senha_atual (string, obrigatório): Senha atual do usuário
  • nova_senha (string, obrigatório): Nova senha do usuário
// Exemplo de requisição { "email": "usuario@exemplo.com", "senha_atual": "senha123", "nova_senha": "novaSenha456" } // Resposta de sucesso { "status": "sucesso", "mensagem": "Senha alterada com sucesso." } // Respostas de erro possíveis { "status": "erro", "mensagem": "Dados para alteração de senha inválidos." } { "status": "erro", "mensagem": "Senha atual incorreta." }

POST /plugins/produtos/controllers/api_produtos_controller.php

  • nome (string, obrigatório): Nome do produto

    • quantidade_estoque (integer): Quantidade em estoque

Filtro Tipo Descrição

marca string Marca do produto (busca exata)
categoria integer ID da categoria
tamanho string Tamanho da variação do produto (busca exata)
cor string Cor da variação do produto (busca exata)
material string Material da variação do produto (busca exata)

variacao_preco_max number Preço máximo das variações
em_estoque boolean Se verdadeiro, filtra produtos disponíveis em estoque
ordenar_por string Campo para ordenação: 'preco', 'nome' ou 'data'
ordem string Direção da ordenação: 'ASC' ou 'DESC'
pagina integer Número da página para paginação (padrão: 1)
limite integer Número de resultados por página (padrão: 20)

Gerenciamento de Categorias de Produtos

POST /plugins/produtos/controllers/api_categoria_controller.php

Cria uma nova categoria de produto

Parâmetros:
  • acao (string, obrigatório): Valor deve ser "criarCategoria"
  • nome (string, obrigatório): Nome da categoria
  • descricao (string, opcional): Descrição detalhada da categoria

  • id_categoria_pai (integer, opcional): ID da categoria pai para subcategorias
// Exemplo de requisição { "acao": "criarCategoria", "nome": "Eletrônicos", "descricao": "Produtos eletrônicos e tecnológicos", "nivel": 1, "id_categoria_pai": null } // Exemplo de subcategoria { "acao": "criarCategoria", "nome": "Smartphones", "descricao": "Telefones inteligentes", "nivel": 2, "id_categoria_pai": 123 } // Resposta de sucesso { "status": "sucesso", "codigo": 201, "mensagem": "Categoria criada com sucesso", "dados": { "id": 4567 } }
